# ClipStash Next 功能优先 TODO

> 目标：先把 Tauri 版推进到“日常可用、可安全替代旧版一部分工作流”，模块整理暂时降级为只在功能需要时顺手做。

## 当前判断

- 当前整体进度按“可替代旧 Python 版日常使用”估算约 `55%`。
- 阶段 0/1 已基本完成。
- 阶段 2/3 的核心读写、复制、导入能力已经具备，并做过大量真实 UI 验收。
- 主要短板不在底层读写，而在“产品可用闭环”：入口、设置、托盘、快捷键、更新、数据安全可视化、发布替换。
- 暂停继续做纯模块拆分；除非为功能落地必要，不再优先整理 `legacy_data.rs` 这类内部细节。

## P0：先做成可安心使用

### 1. 数据安全面板

- [ ] 在 UI 中显示当前旧库路径、图片目录、DB 是否存在、图片目录是否存在。
- [ ] 显示当前普通/归档/总数、图片关联数、孤立图片数。
- [ ] 显示最近 DB 备份列表，至少包含文件名、时间、大小。
- [ ] 显示最近图片备份目录列表。
- [ ] 提供“打开数据目录”“打开备份目录”按钮。
- [ ] 当只读审计发现基线变化时，在 UI 中提示“旧库可能被其他版本改动过”。
- [ ] 验收：不写 DB；`npm run verify:legacy-readonly` 通过；真实 UI 能看到当前路径和备份。

### 2. 日常主流程打磨

- [ ] 统一新增/编辑/删除/归档/恢复/复制/导入后的成功提示样式。
- [ ] 统一失败提示，失败时保留弹窗状态，不丢输入。
- [ ] 新增/编辑弹窗里明确显示本次操作会写旧库，并展示备份策略。
- [ ] 删除弹窗突出显示“会备份 DB 和图片后删除”，避免误删真实历史数据。
- [ ] 列表刷新后保持当前视图、排序和滚动位置尽量不跳。
- [ ] 验收：创建临时消息、编辑、归档、恢复、删除全流程可通过 UI 完成并清理。

### 3. 图片与列表体验

- [ ] 多图消息的展开/收起交互可用，不出现布局跳动或按钮挤压。
- [ ] 图片缺失时显示明确占位和文件名，不出现空白块。
- [ ] 图片预览支持上一张/下一张。
- [ ] 已归档列表大量消息时保持可滚动、不卡死。
- [ ] 验收：使用包含多图、缺图、已归档大列表的旧库做 UI 检查。

## P1：替代旧版的系统能力

### 4. 设置页

- [ ] 读取旧 `settings.json`，缺失字段提供默认值。
- [ ] 保存设置时由 Rust 写入，并返回实际生效值。
- [ ] 设置项至少覆盖：窗口置顶、导入后归档、粘贴间隔、快捷键、开机启动。
- [ ] 保存失败时显示明确错误，不伪装成成功。
- [ ] 验收：保存后重启 Next，设置仍生效；旧 Python 不因新增字段崩溃。

### 5. 托盘与窗口行为

- [ ] 实现托盘驻留。
- [ ] 托盘菜单包含：显示/隐藏主窗口、打开数据目录、退出。
- [ ] 关闭窗口时按设置决定隐藏到托盘或退出。
- [ ] 验收：托盘菜单可操作，退出后进程结束。

### 6. 全局快捷键

- [ ] 支持设置并注册全局呼出快捷键。
- [ ] 快捷键冲突时显示错误。
- [ ] 重启后快捷键仍生效。
- [ ] 验收：从其他窗口按快捷键能呼出 ClipStash Next。

### 7. 剪贴板监听和快速保存

- [ ] 明确是否需要替代旧版自动保存剪贴板能力。
- [ ] 如需要，实现文字/图片剪贴板读取预览。
- [ ] 支持从剪贴板快速保存文字、图片、图文。
- [ ] 验收：复制文字/图片后可通过 Next 保存到旧库，旧 Python 可读。

## P2：发布替换准备

### 8. 更新检查

- [ ] 实现 GitHub Release 检查。
- [ ] 检查失败时显示失败原因。
- [ ] 检查失败或有更新时提供可点击 Release 页面入口。
- [ ] 验收：断网/失败场景不会只有“待实现”或模糊提示。

### 9. 安装包和回滚

- [ ] 固化 `npm run tauri build` 的发布产物路径和命名。
- [ ] 安装包默认使用旧数据目录。
- [ ] 首次启动执行只读审计并提示结果。
- [ ] 明确旧 Python 版回滚方法。
- [ ] 验收：安装后可读取旧库；旧 Python 仍能打开同一旧库。

### 10. 真实验收数据集

- [ ] 准备或生成回归测试 DB，覆盖纯文字、单图、2-4 图、18 图、已归档、纯图片、长文本、缺失图片。
- [ ] 避免反复直接用真实用户库做破坏性验收。
- [ ] 将回归数据集路径和使用方式写入文档。
- [ ] 验收：CI 或本地脚本可对测试 DB 跑只读一致性检查。
