# 阶段 2/3 真实 UI 验收清单

## 目标

- 补齐浏览器 mock 测试之外的应用级点击验收。
- 验证 Tauri WebView 中的真实按钮、弹层、文件选择、剪贴板和窗口选择流程。
- 保持旧 `clipstash.db`、旧 `images/` 和旧 Python 版兼容。

## 通用前置条件

- 开始前运行 `npm run verify:legacy-readonly`，记录 `normal`、`archived`、`total`、`joined_images`、`orphan_images`。
- 确认测试消息：
  - 图文消息：`id=114`，未归档，文字 52 字符，1 张图片。
  - 图片消息：`id=113`，1 张图片。
- 每个会写 DB 或图片目录的 UI 验收前，确认自动备份路径会在结果 UI 中展示。
- 每个验收后再运行 `npm run verify:legacy-readonly`。
- 涉及旧 Python 兼容时，使用旧版 `db.py` 或旧应用读取新写入/改动后的消息。

## 阶段 2：写入 UI 验收

### 图片/图文新增入口

1. 启动 Tauri dev 或打包应用，打开 ClipStash Next。
2. 在“新增图片 / 图文消息”中选择一张 1x1 测试图片。
3. 不填写文字，勾选写入确认，点击“新增并备份”。
4. 记录新消息 id、备份路径和图片文件名。
5. 用旧 Python 读取该消息，确认 `message_images` 文件存在且可读。
6. 再执行一次带文字的图文新增，确认文字和图片均可被旧 Python 读取。

### 编辑/删除入口

1. 选择专用测试消息，不使用用户真实重要消息。
2. 编辑文字后确认保存，记录 DB 备份路径。
3. 替换图片后确认旧图片已备份，新图片被旧 Python 读取。
4. 删除专用测试消息前确认备份路径；删除后验证 DB 中消息和图片关联消失。
5. 删除验收只对本轮创建的测试消息执行，不对历史真实消息执行。

## 阶段 3：复制与导入 UI 验收

### 文字复制

1. 选择 `id=114`，点击“复制文字”。
2. 用旧 Python `clipboard_utils.get_clipboard_text()` 读回剪贴板。
3. 确认读回文字长度为 52，旧库审计计数不变。

### 图片复制

1. 选择 `id=114` 的可读图片，点击“复制图片”。
2. 用旧 Python `clipboard_utils.get_clipboard_image()` 读回剪贴板。
3. 确认尺寸为 `1x1`，旧库审计计数不变。

### 导入队列与受控粘贴

1. 启动临时 WinForms 文本/图片/队列目标窗口。
2. 在 `id=114` 上点击“查看队列”，确认队列顺序为文字优先、图片其次。
3. 刷新目标窗口，选择目标并校验。
4. 先执行“粘贴第 1 项”，读回目标窗口文字。
5. 再执行图片项粘贴，读回目标窗口图片尺寸。
6. 执行“粘贴整队列”，读回文字和图片。
7. 默认不勾选归档时，确认旧库审计计数不变。
8. 勾选“整队列粘贴成功后归档旧库消息”时，只对测试消息执行；记录 DB 备份路径，验收后恢复测试消息为未归档。

## 回滚

- 代码回滚：使用对应 git commit 的 `git revert`。
- DB 回滚：关闭新旧应用，将目标 `clipstash.db.bak-*` 复制回 `clipstash.db`。
- 图片回滚：用验收记录中的图片文件名删除本轮新增图片；替换/删除场景优先使用自动生成的图片备份目录恢复。
- 外部窗口粘贴不可自动回滚，目标应用中使用撤销或关闭临时目标窗口。

## 暂缓项

- 内置浏览器截图验收曾遇到 `net::ERR_BLOCKED_BY_CLIENT`，暂不重复同一路径。
- 若需要视觉证据，优先使用 Tauri 桌面窗口人工截图或后续接入桌面自动化工具。
